 36: /**
 37:  * Holds the parameters used for exponential backoff logic.
 38:  *
 39:  * The "total timeout" parameter has ultimate control over how long the logic should
 40:  * keep trying the remote call until it gives up completely. The higher the total
 41:  * timeout, the more retries can be attempted. The other settings are considered
 42:  * more advanced.
 43:  *
 44:  * Retry delay and timeout start at specific values, and are tracked separately from
 45:  * each other. The very first call (before any retries) will use the initial timeout.
 46:  *
 47:  * If the last remote call is a failure, then the retrier will wait for the current
 48:  * retry delay before attempting another call, and then the retry delay will be
 49:  * multiplied by the retry delay multiplier for the next failure. The timeout will
 50:  * not be affected, except in the case where the timeout would result in a deadline
 51:  * past the total timeout; in that circumstance, a new timeout value is computed
 52:  * which will terminate the call when the total time is up.
 53:  *
 54:  * If the last remote call is a timeout, then the retrier will compute a new timeout
 55:  * and make another call. The new timeout is computed by multiplying the current
 56:  * timeout by the timeout multiplier, but if that results in a deadline after the
 57:  * total timeout, then a new timeout value is computed which will terminate the call
 58:  * when the total time is up.
 59:  */
